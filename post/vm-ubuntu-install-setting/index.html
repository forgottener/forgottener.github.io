虚拟机安装ubuntu及其配置

2014-01-26
linux

配置虚拟机

#安装过程修改源
http://mirrors.aliyun.com/ubuntu

#远程控制依赖检查安装
sudo apt-get update
sudo apt-cache policy openssh-client openssh-server
sudo apt-get install openssh-client

#设置root密码
sudo passwd root

#允许root登录
vi /etc/ssh/sshd_config

找到下方注释注释内容修改
#PermitRootLogin prohibit-password
PermitRootLogin yes

#重启ssh服务使之生效
service ssh restart

同步时间

  • 设置时区
dpkg-reconfigure tzdata

#选择Asia（亚洲）-> Shanghai（上海）
  • 时间同步
# 安装 ntpdate
apt-get install ntpdate

# 设置系统时间与网络时间同步（cn.pool.ntp.org 位于中国的公共 NTP 服务器）
ntpdate cn.pool.ntp.org

# 将系统时间写入硬件时间
hwclock --systohc
  • 确认时间
date

# 输出如下（自行对照与系统时间是否一致）
Wed May 20 09:02:29 CST 2020

修改ip和dns

修改主机名

  • 修改cloud.cfg防止重启后主机名还原
vi /etc/cloud/cloud.cfg

#该配置默认为false,修改为true即可
preserve_hostname: true
  • 修改主机名
#修改主机名
hostnamectl set-hostname deployment

#配置host
cat >> /etc/hosts << EOF
192.168.141.130 deployment
EOF
  • 修改ip 编辑vi /etc/netplan/50-cloud-init.yaml配置文件，修改内容如下：
network:
  ethernets:
    enp1s0:
      addresses: [192.168.141.130/24]
      gateway4: 192.168.141.2
      nameservers:
        addresses: [192.168.141.2]
  version: 2
  • 修改dns
vi /etc/systemd/resolved.conf

# 第 15 行
# 取消 DNS 行注释，并增加 DNS 配置如：114.114.114.114，修改后重启下计算机
DNS=114.114.114.114
